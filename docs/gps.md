# AliveWorld GPS — техническая документация

## API-ограничения Luanti 5.16.1

### `hud_elem_type = "minimap"` — существует
Сервер может создать HUD-элемент миникарты. Поддерживаемые поля:
`position`, `size` (включая отрицательные проценты), `alignment`, `offset`, `z_index`.

Форма (круг/квадрат) задаётся ТОЛЬКО клиентом через `Shift+V`.
Сервер не имеет API для принудительной установки круглой формы.

### `player:set_minimap_modes()` — существует
Позволяет задать набор режимов миникарты. Каждый режим:
```lua
{
    type = "off" | "surface" | "radar" | "texture",
    label = "строка",
    size = <число>,          -- сторона в блоках (1-512)
    texture = "file.png",    -- только для "texture"
    scale = <число>,         -- только для "texture"
}
```

### Сервер НЕ может
- Узнать текущий режим/форму миникарты игрока
- Принудительно переключить форму (круг/квадрат)
- Получить размер экрана игрока в пикселях

### `player:set_look_horizontal() / get_look_horizontal()`
Сервер может читать и устанавливать yaw игрока.

### `player:hud_set_flags({minimap = false})`
Отключает встроенную миникарту (builtin), но НЕ влияет на HUD-элемент
миникарты, добавленный сервером через `hud_add`.

## Решение AliveWorld GPS

1. Включение GPS создаёт серверный `type = "minimap"` HUD-элемент
2. Встроенная миникарта скрывается через `hud_set_flags`
3. `set_minimap_modes` задаёт режимы с известным размером в блоках
4. Маркеры — отдельные `type = "image"` HUD-элементы поверх миникарты
5. Позиция маркеров пересчитывается с учётом yaw игрока
6. Масштаб: известный размер миникарты в пикселях ÷ размер в блоках
7. Обновление: change-driven (глобальный шаг только для детекции движения)
8. При выключении GPS: восстанавливается `hud_set_flags({minimap = true})`
